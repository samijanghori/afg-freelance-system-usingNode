Node.js چیست؟

Node.js یک محیط اجرای جاوااسکریپت در سمت سرور است که بر پایه موتور V8 گوگل کروم ساخته شده. این فناوری به توسعه‌دهندگان اجازه می‌دهد تا با استفاده از جاوااسکریپت، برنامه‌های back-end، API‌ها، ابزارهای خط فرمان و حتی دسکتاپ بسازند.

📌 مبتنی بر مدل رویداد محور و ورودی/خروجی غیرهمزمان → ایده‌آل برای برنامه‌های بلادرنگ و پرترافیک.
⚙️

ویژگی‌های کلیدی

  • ✅ غیرهمزمان (Asynchronous) و Non-blocking I/O
  • ✅ مدیریت بسته NPM با بیش از ۲ میلیون کتابخانه
  • ✅ سرعت بالا به لطف موتور V8 (کامپایل لحظه‌ای)
  • ✅ مقیاس‌پذیری با استفاده از معماری رویدادمحور و خوشه‌بندی
  • ✅ Single-threaded اما کارآمد با Event Loop
  • ✅ پشتیبانی از WebSocket، REST، GraphQL و میکروسرویس‌ها
🏗️

کاربردهای محبوب

  • 🔹 توسعه APIهای سریع RESTful و Backend
  • 🔹 برنامه‌های بلادرنگ (چت، نوتیفیکیشن، گیم)
  • 🔹 ابزارهای خط فرمان (CLI)
  • 🔹 میکروسرویس‌ها و معماری سرورلس (Serverless)
  • 🔹 استریمینگ داده و برنامه‌های پخش ویدئو
  • 🔹 فریمورک‌های معروف: Express، Nest.js، Fastify، Koa

💻 یک سرور ساده با Node.js (بدون فریمورک)

فقط با چند خط کد، یک وب‌سرور HTTP ایجاد کنید:

const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/html; charset=utf-8'});
  res.end('<h1>سلام از دنیای Node.js! 🚀</h1><p>اجای جاوااسکریپت در سمت سرور</p>');
});

server.listen(3000, () => {
  console.log('سرور روی پورت 3000 در حال اجراست');
});

✏️ کد بالا را ذخیره کنید (مثلا server.js) و با دستور node server.js اجرا کنید. سپس مرورگر: http://localhost:3000

📈

مزیت رقابتی Node.js

Node.js با اشتراک‌گذاری زبان جاوااسکریپت بین فرانت‌اند و بک‌اند، تیم‌های توسعه را چابک‌تر می‌سازد. حلقه رویداد (event loop) به آن اجازه می‌دهد هزاران درخواست همزمان را با مصرف حافظه نسبتاً کم مدیریت کند. همچنین NPM بزرگترین مخزن متن‌باز جهان است.

⚠️

نکات مهم / محدودیت‌ها

  • 🔸 به دلیل ماهیت single-thread، تسک‌های سنگین CPU (تصویربرداری، رمزنگاری سنگین) می‌توانند حلقه رویداد را مسدود کنند.
  • 🔸 کالبک‌های تو در تو (callback hell) – اما با استفاده از async/await و promiseها این مشکل حل شده.
  • 🔸 برای برنامه‌های با محاسبات سنگین، بهتر است از Worker Threads یا معماری مبتنی بر میکروسرویس استفاده کنید.
